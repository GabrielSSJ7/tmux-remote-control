<objective>
Replace permissive CORS with explicit origin allowlist configured in config.toml.

Purpose: `CorsLayer::permissive()` allows any website to make cross-origin requests to the backend. This is a security risk if the server is exposed beyond localhost. After this plan, only origins explicitly listed in `config.toml` are allowed, and an empty list rejects all cross-origin requests with a logged warning.

Output: New `allowed_origins` field in `ServerConfig`, `create_router` takes origins parameter for CORS configuration, existing config.toml backward-compatible via `#[serde(default)]`, integration test stubs for CORS.
</objective>

```rust
pub fn create_router(allowed_origins: &[String]) -> Router<Arc<AppState>> {
```

2. Build the CORS layer from the origins list:
```rust
use tower_http::cors::{CorsLayer, AllowOrigin};
use axum::http::{HeaderValue, Method, header};

if allowed_origins.is_empty() {
    tracing::warn!("allowed_origins is empty -- all cross-origin requests will be rejected");
}

let origins: Vec<HeaderValue> = allowed_origins
    .iter()
    .filter_map(|o| o.parse().ok())
    .collect();

let cors = CorsLayer::new()
    .allow_origin(AllowOrigin::list(origins))
    .allow_methods([Method::GET, Method::POST, Method::PUT, Method::DELETE])
    .allow_headers([header::CONTENT_TYPE, header::AUTHORIZATION]);
```

3. Replace `.layer(tower_http::cors::CorsLayer::permissive())` with `.layer(cors)`.

4. Keep `.layer(tower_http::trace::TraceLayer::new_for_http())` as the outermost layer (registered AFTER cors, which means cors wraps before trace in Axum's reverse-order convention). The current order is cors then trace -- keep it exactly as is. Per RESEARCH.md Pitfall 3: CorsLayer must be registered BEFORE TraceLayer (applied after) to intercept preflight before trace logs fire.

5. In `main.rs`: update the call from `create_router()` to `create_router(&config.server.allowed_origins)`. The `config` variable is available before `state` is constructed.
