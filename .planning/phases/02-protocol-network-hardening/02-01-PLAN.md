<objective>
Migrate backend WebSocket authentication from URL query parameter to first-frame binary protocol and add rate limiting to the WebSocket upgrade path.

Purpose: The auth token currently appears in the WebSocket URL (`?token=...`), which leaks into HTTP logs, proxy logs, and Wireshark captures. Moving auth to a first-frame binary message (0x05) eliminates this exposure. Rate limiting the upgrade path prevents connection floods from exhausting server resources before auth is even attempted.

Output: Modified `protocol.rs` with `Frame::Auth` variant, rewritten `ws_handler` and `handle_socket` in `terminal.rs` with first-frame auth handshake and pre-upgrade rate limiting, integration test stubs for WS auth.
</objective>

**In `terminal.rs`:**

1. Remove the `WsQuery` struct and the `Query` import entirely.

2. Add these imports:

   - `use tokio::time::{timeout, Duration};`

3. Rewrite `ws_handler` signature to:
```rust
pub async fn ws_handler(
    ws: WebSocketUpgrade,
    State(state): State<Arc<AppState>>,
    Path(session_id): Path<String>,
    ConnectInfo(addr): ConnectInfo<SocketAddr>,
) -> Result<impl IntoResponse, AppError>
```

4. In `ws_handler` body: call `state.rate_limiter.check(addr.ip())` first. If false, return `Err(AppError::RateLimited)`. Then check `TmuxManager::session_exists`. Then call `ws.on_upgrade(move |socket| handle_socket(socket, state, session_id, addr.ip()))`. Remove all token validation from this function (it moves into handle_socket).

5. Change `handle_socket` signature to:
```rust
async fn handle_socket(mut socket: WebSocket, state: Arc<AppState>, session_id: String, ip: IpAddr)
```

6. At the top of `handle_socket`, BEFORE the existing PTY setup code, add the first-frame auth handshake:
   - Use `timeout(Duration::from_secs(10), socket.recv())` to read the first message.
   - On timeout, `Ok(Err(_))`, or `Ok(None)`: close socket and return.
   - On `Ok(Some(Ok(Message::Binary(data))))`: decode with `Frame::decode(&data)`.
   - If `Frame::Auth(token)`: do constant-time comparison against `state.config.auth.token.as_bytes()`. On mismatch, call `state.rate_limiter.record_failure(ip)`, send a SessionEvent frame with `event_type: "unauthorized"`, close socket, return.
   - If any other frame type: send SessionEvent "unauthorized", close socket, return.
   - On successful auth: call `state.rate_limiter.reset(ip)` and continue to the existing PTY setup code.

7. The rest of `handle_socket` (PTY setup, main loop, cleanup) remains unchanged.
