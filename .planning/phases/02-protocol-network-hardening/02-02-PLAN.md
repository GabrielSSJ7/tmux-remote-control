<objective>
Migrate Android WebSocket client from URL-based token authentication to first-frame binary auth protocol.

Purpose: The Android app currently sends the auth token as `?token=...` in the WebSocket URL. This leaks the token into OkHttp logs, device proxy logs, and network captures. After this plan, the URL contains no token and auth is sent as a binary `Frame.Auth` message immediately after connection, matching the new backend first-frame handshake.

Output: Updated `Protocol.kt` with `Frame.Auth` variant, rewritten `TerminalSocket.kt` with token-free URLs and first-frame auth in `onOpen`.
</objective>

**Structural changes to TerminalSocket.kt:**

1. Replace `private var currentUrl: String? = null` with three separate fields:
```kotlin
private var currentBaseWsUrl: String? = null
private var currentSessionId: String? = null
private var currentToken: String? = null
```

2. Rewrite `connect()`:
   - Normalize the base URL and compute `wsBase` exactly as before (http->ws, https->wss).
   - Build `wsUrl` as `"$wsBase/sessions/$sessionId/terminal"` -- NO `?token=` parameter.
   - Store `currentBaseWsUrl = wsBase`, `currentSessionId = sessionId`, `currentToken = token`.
   - Call `doConnect(wsUrl, token)`.

3. Change `doConnect` signature to `private fun doConnect(url: String, token: String)`:
   - In the `WebSocketListener.onOpen` callback, BEFORE the existing `_state.value = ConnectionState.CONNECTED` line, add:
     ```kotlin
     webSocket.send(Frame.Auth(token.toByteArray()).encode().toByteString())
     ```
   - The rest of `onOpen` remains unchanged (`_state.value = CONNECTED`, `reconnector.reset()`, `consecutivePingFailures = 0`, `startPing()`).

4. Rewrite `scheduleReconnect()`:
   - Instead of using `currentUrl`, reconstruct the URL from stored fields:
     ```kotlin
     val wsBase = currentBaseWsUrl ?: return
     val sessionId = currentSessionId ?: return
     val token = currentToken ?: return
     val url = "$wsBase/sessions/$sessionId/terminal"
     reconnectJob = scope.launch {
         delay(reconnector.nextDelay())
         doConnect(url, token)
     }
     ```

**Key pitfalls to avoid (from RESEARCH.md):**
- Do NOT store `?token=` in any URL field (Pitfall 2). The token is stored separately in `currentToken`.
- Send `Frame.Auth` BEFORE `startPing()` in `onOpen` -- the server expects auth as the very first frame.
- Use `toByteString()` (OkHttp's `ByteString`) for sending binary frames, matching the existing `send()` method pattern.
